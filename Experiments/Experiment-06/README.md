# Experiment 06 - Functions, Recursion, and Lambda

## Objective
Practice defining and using **functions**, basic **recursion**, and simple
**lambda** expressions.

## Prerequisites
- Functions: parameters, return values
- Lists/dictionaries
- Recursion basics (base case + recursive step)

## Exercises (Scripts)
1. `scripts/01_max_min.py` - Find max/min without built-ins
2. `scripts/02_sum_cubes.py` - Sum of cubes for values smaller than `n`
3. `scripts/03_recursive_print.py` - Print 1..n using recursion
4. `scripts/04_recursive_fib.py` - Fibonacci series using recursion
5. `scripts/05_cone_volume.py` - Cone volume using lambda
6. `scripts/06_lambda_max_min.py` - Return (max, min) using lambda
7. `scripts/07_function_arguments.py` - Default/keyword/*args/**kwargs demo
8. `scripts/08_dict_values_same.py` - Check if all dict values are same
9. `scripts/09_lists_to_dict.py` - Create dict using `zip`

## Explanations (PDF)
- Compiled PDFs are in `explanations/`
- LaTeX sources are in `explanations/latex/`

## Evaluation Rubric (Suggested)
- Correctness: 4
- Function design (reuse, parameters): 2
- Recursion correctness (base case): 2
- Output formatting: 1
- Viva: 1

## Viva Questions (Sample)
1. What is a base case and why is it required in recursion?
2. Why is recursive Fibonacci inefficient without memoization?
3. Difference between `*args` and `**kwargs`?
4. When should you avoid recursion in Python?
5. When is a lambda helpful and when does it reduce readability?
